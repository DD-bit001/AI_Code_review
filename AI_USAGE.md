# AI Usage Report

## 1. AI Tools Used
During the development of this full-stack application, the following AI tools were utilized:
- **Gemini 3.1 Pro (High)**: Used as the primary agentic assistant for boilerplate generation, logic scaffolding, and architectural refinement.

## 2. Prompts Used
Prompts provided to the AI system focused on:
- "Set up a FastAPI backend with SQLAlchemy models representing Users, Projects, Files, AI Providers, Reviews, and Chat Sessions."
- "Write an OpenAI-compatible API wrapper function in Python that dynamically takes a base_url and an api_key to fetch code reviews."
- "Generate a Next.js 16 UI using Tailwind CSS that looks premium, modern, and implements a multi-tab view for Code Explorer, Reviews, and Chat."

## 3. Generated Code
The following parts of the codebase were heavily assisted or generated by AI:
- The initial FastAPI folder structure and boilerplate (e.g. `main.py`, `database.py`, `models.py`).
- The JWT authentication flow in `auth.py` and `auth_router.py`.
- The Tailwind CSS landing page design and visual elements in `app/page.tsx` (e.g., gradients, component layouts, Lucide icons).
- The dynamic interaction with the `AsyncOpenAI` SDK in `ai.py`.

## 4. Manually Written/Modified Code
The following aspects required manual intervention, structural integration, and decision-making:
- **State Management & Interceptors**: Hooking up Zustand in the Next.js app (`store.ts`) and making sure Axios instances append JWT tokens seamlessly.
- **ZIP File Processing logic**: Manually refining the ZIP extraction in `project_router.py` to ensure robust error handling and file filtering (skipping binary files, ignoring directories).
- **Component Stitching**: Wiring up the frontend `ProjectPage` tabs to properly fetch code contexts, manage loading states, and update the UI in response to backend asynchronous processing.
- **Database Fallbacks**: Injecting logical branches to allow fallback to SQLite if PostgreSQL isn't available, maintaining developer convenience.

## 5. Engineering Decisions
- **FastAPI over NestJS**: While NestJS was preferred, FastAPI was selected to optimize iteration speed and simplify the native integration of Python-based AI SDKs (`openai`). Python provides unparalleled ecosystem support for generative AI features.
- **Client-Side File Parsing**: Rather than processing giant repositories entirely in memory, files are filtered during ZIP decompression to ignore binaries (`.png`, `.exe`, etc.), ensuring the database isn't overloaded and context windows aren't bloated with unreadable data.
- **Abstracted AI Providers**: Implemented a configurable settings model that overrides standard OpenAI endpoints with custom user-provided URLs. This guarantees 100% compatibility with local solutions like `LM Studio` and `Ollama` without any code changes.
